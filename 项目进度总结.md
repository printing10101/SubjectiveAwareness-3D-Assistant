## 一、项目概览

### 1.1 项目目标
构建一套基于大语言模型的帮信罪主观明知智能分析系统，实现案件材料自动解析、三维度智能分析、可视化报告生成等核心功能，辅助检察官快速、一致地完成"主观明知"要素的审查判断。

### 1.2 技术栈
- **后端**: FastAPI + SQLAlchemy + SQLite/PostgreSQL
- **前端**: Vue 3 + Pinia + Vue Router + Vite
- **AI模型**: Ollama (deepseek-r1:7b)
- **数据库**: SQLite (开发) / PostgreSQL (生产)

## 六、已知问题与限制

### 6.1 技术债务
1. **后端类型注解不完整**: 286 个 mypy 错误，主要分布在 pipeline.py（78 个错误）等核心模块
2. **测试覆盖率偏低**: 整体覆盖率 36%，部分核心模块覆盖率不足
3. **Ruff 检查问题**: 226 个问题需手动修复，主要是类型注解和代码格式问题

### 6.2 功能限制
1. **暂不支持批量案件处理**: 当前架构设计为单案件串行分析，批量处理需要引入任务队列
2. **暂不支持多模型并行对比**: Ollama 本地部署资源有限，并行推理会导致显存不足
3. **暂不支持实时协作标注**: 标注系统为单人模式，多人协作需要引入 WebSocket 实时通信
4. **暂不支持自定义分析维度**: 三维度分析模型经过司法专家验证，自定义维度可能影响分析结果的法律效力
5. **暂不支持离线部署**: 系统依赖 Ollama 服务进行模型推理，完全离线需要本地部署完整模型服务栈
