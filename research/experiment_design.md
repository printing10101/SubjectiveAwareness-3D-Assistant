# 回溯性对比实验方案：AI辅助分析对"主观明知"认定的影响研究

---

## 1. 实验背景与意义

### 1.1 司法实践中"主观明知"认定的挑战

在帮助信息网络犯罪活动罪（以下简称"帮信罪"）等涉信息网络犯罪的司法实践中，"主观明知"的认定长期面临以下突出挑战：

- **主观认定的不确定性**：行为人的主观心理状态难以通过直接证据加以证明，司法人员只能依赖外部客观行为来推定主观明知，推定的准确性和一致性高度依赖于个人经验。
- **裁判标准不统一**：不同法官对类似案件事实的"主观明知"认定可能出现分歧，导致同案不同判现象，影响司法公信力。
- **案件数量激增**：近年来帮信罪案件数量大幅增长，司法人员面临巨大的办案压力，有限的时间和精力可能影响分析判断的审慎性和全面性。
- **证据类型复杂化**：网络犯罪涉及电子数据、聊天记录、资金流水等多种类型证据，分析维度多、关联关系复杂，传统分析方法难以高效整合多元证据信息。

### 1.2 AI辅助分析的潜在价值

人工智能技术，尤其是自然语言处理和大语言模型的发展，为司法领域的"主观明知"认定提供了新的可能路径：

- **标准化分析框架**：AI辅助工具可基于大量已决案件和法律规范，构建标准化的证据分析框架，减少个人经验差异带来的认定偏差。
- **一致性提升**：通过统一的算法和分析逻辑，AI辅助工具能够帮助不同司法人员在同一案件的事实认定上趋于一致。
- **效率提升**：AI可以快速梳理和提炼案件材料中的关键证据信息，生成结构化的分析报告，大幅缩短案件分析时间。
- **决策参考**：AI分析结果可以为司法人员提供客观、可追溯的参考依据，辅助而非替代人类判断，最终决策权仍由司法人员行使。

### 1.3 研究意义

本实验旨在通过科学、严谨的回溯性对比实验设计，系统评估AI辅助分析工具对司法人员认定"主观明知"的一致性和效率的实际影响，为AI技术在司法领域的推广应用提供坚实的实证依据，同时也为相关技术产品的优化方向提供数据支撑。

---

## 2. 实验目的

本实验的核心目的包括：

1. **验证一致性提升效果**：系统检验AI辅助分析工具是否能显著提升不同司法人员对同一案件"主观明知"认定的判定一致性水平（以Cohen's Kappa系数为衡量指标）。
2. **量化效率改善程度**：精确测量并比较两组人员完成单个案件"主观明知"分析所需的平均时间，评估AI辅助对分析效率的实际影响。
3. **评估AI判准率**：将AI分析工具的结论与法院生效判决进行对比，计算AI结论与判决的吻合度，评估AI分析结果的有效性。
4. **探索影响因素**：分析案件难度、证据完整度、司法人员从业年限等变量对AI辅助效果的影响，识别AI辅助工具的适用场景和边界条件。

---

## 3. 实验方法

### 3.1 实验设计类型

采用**回溯性对比实验设计**（Retrospective Controlled Experiment Design），即选取已审结的帮信罪案件材料，由两组司法人员在不同的分析条件下分别进行"主观明知"认定判断。

### 3.2 实验逻辑框架

| 要素 | 说明 |
|------|------|
| 研究类型 | 回溯性对比实验 |
| 研究对象 | 已审结帮信罪案件（已脱敏处理） |
| 分组方式 | 独立分组（Between-subjects design） |
| 自变量 | AI辅助分析报告（有/无） |
| 因变量 | 认定一致性（Kappa系数）、分析耗时（分钟） |
| 控制变量 | 案件难度、案件类型、证据完整度 |
| 随机化 | 案件随机分组、评估人员随机分组 |

### 3.3 实验假设

- **H1**：B组（实验组）的案件认定一致性Kappa系数显著高于A组（对照组），且≥0.65。
- **H2**：B组（实验组）完成单个案件分析的平均耗时显著低于A组（对照组）。
- **H3**：AI分析结论与法院生效判决的吻合度达到较高水平（≥80%）。

## 4. 实验对象

### 4.1 案件来源

选取**贵州法院系统**已审结的帮助信息网络犯罪活动罪案件，具体来源涵盖贵州省内各级人民法院（包括但不限于贵阳市、遵义市、六盘水市等地方法院）。

### 4.2 案件数量

- 总案件数：**20-30件**
- 具体数量根据案件材料的完整度和可用性最终确定
- 确保案件涵盖不同的"主观明知"认定情形（包括认定明知、不认定明知、部分认定等）

### 4.3 案件筛选标准

| 标准类型 | 具体要求 |
|----------|----------|
| 纳入标准 | ① 案件已审结并生效；② 案件涉及"主观明知"认定的争议；③ 案件材料完整，包含起诉书、判决书、主要证据材料；④ 案件事实清晰，能够形成完整的证据链 |
| 排除标准 | ① 案件材料严重缺失或不完整；② 涉及国家秘密或敏感信息无法充分脱敏；③ 案件事实过于特殊，缺乏普遍代表性 |

### 4.4 脱敏处理要求

对所有案件材料进行严格的脱敏处理，具体包括：

- 当事人姓名、身份证号、住址、联系方式等个人身份信息进行替换或删除
- 证人、被害人等诉讼参与人身份信息脱敏
- 案件涉及的具体银行账号、IP地址、设备MAC地址等设备/账户信息模糊化处理
- 可能识别出具体办案人员的信息进行脱敏
- 脱敏处理后由数据安全专员进行审核，确保符合《个人信息保护法》《数据安全法》等相关法律法规要求

## 5. 实验分组

### 5.1 分组设计

| 组别 | 组名 | 实验条件 | 人数 |
|------|------|----------|------|
| **A组** | **对照组** | 仅依靠个人专业经验和传统分析方法，自行查阅案件材料，对案件"主观明知"进行独立判断 | 10-15人 |
| **B组** | **实验组** | 在个人专业经验基础上，参考AI辅助分析工具生成的标准化分析报告，结合案件材料对"主观明知"进行独立判断 | 10-15人 |

### 5.2 分组原则

- **随机分配原则**：符合条件的司法人员通过随机数生成器进行随机分组，确保两组人员在从业年限、专业背景等特征上无显著差异。
- **独立判断原则**：两组人员之间不得进行交流讨论，A组人员不得接触AI分析报告，B组人员必须参考AI报告进行判断。
- **交叉验证**：每件案件至少由同一组内的3名司法人员分别进行独立判断，以计算组内一致性。

### 5.3 双盲设计说明

- 参与实验的司法人员**知晓**自己属于哪一组（因操作条件不同，无法实现完全盲法）
- 案件分配时，两组使用的案件集合保持一致，但个体分配的案件顺序随机化
- AI分析报告的结论不会直接给出"明知/不明知"的二元结论，而是提供证据梳理、风险因素分析等结构化信息，由司法人员自行判断

## 6. 评估指标

### 6.1 认定一致性（Kappa系数）

- **指标定义**：使用Cohen's Kappa系数衡量同一案件在不同评估人员之间的认定一致程度。
- **计算公式**：

  ```
  κ = (Po - Pe) / (1 - Pe)
  ```

  其中，Po为观察一致率，Pe为期望一致率（随机一致概率）。

- **评判标准**：

  | Kappa值范围 | 一致性程度 |
  |-------------|------------|
  | κ < 0.00 | 低于随机一致性 |
  | 0.00 ≤ κ < 0.20 | 一致性极低 |
  | 0.20 ≤ κ < 0.40 | 一致性较低 |
  | 0.40 ≤ κ < 0.60 | 中等一致性 |
  | 0.60 ≤ κ < 0.80 | 良好一致性 |
  | 0.80 ≤ κ ≤ 1.00 | 高度一致性 |

- **目标要求**：实验组（B组）Kappa值 **≥ 0.65**，视为AI辅助条件下具有良好的一致性。

### 6.2 分析耗时

- **指标定义**：从评估人员开始阅读案件材料到完成"主观明知"认定判断并提交结果之间的总时间。
- **记录方式**：通过实验平台自动计时，精确到秒，后续统计时转换为分钟并保留一位小数。
- **统计口径**：
  - 单件案件平均耗时（分钟）= 该组所有人员完成所有案件的总耗时 / （人数 × 案件数）
  - 对比两组平均耗时的差异并进行独立样本t检验

### 6.3 AI与判决一致率

- **指标定义**：AI辅助分析工具生成的"主观明知"分析结论与法院实际生效判决的吻合度。
- **计算方式**：

  ```
  一致率（%）= （AI结论与判决一致的案件数 / 总案件数） × 100%
  ```

- **精确度**：计算结果精确到整数位。
- **分类统计**：
  - AI结论与判决完全一致的案件比例
  - AI结论与判决部分一致（如认定一致但理由不同）的案件比例
  - AI结论与判决不一致的案件比例
  - 对不一致案件进行定性分析，识别AI判断的偏差模式

### 6.4 辅助分析指标（可选）

- **置信度评分**：每位评估人员在提交认定结论时，同步给出该结论的置信度评分（1-5分制）。
- **分析满意度**（仅B组）：实验组人员对AI辅助分析报告的质量和有用性进行评分。

## 7. 实验流程

### 7.1 阶段一：案件准备阶段

| 步骤 | 内容 | 负责方 | 时间节点 |

| 2. 材料整理 | 对筛选出的案件材料进行数字化扫描、OCR识别、文本提取和格式统一化处理 | 研究团队 | 第2-3周 |
| 3. 脱敏处理 | 对所有案件材料进行脱敏处理，删除或替换敏感信息 | 数据安全专员 | 第3-4周 |
| 4. 脱敏审核 | 由独立审核人员对脱敏效果进行复核，确保无敏感信息泄露 | 审核人员 | 第4周 |

| 7. 难度均衡化 | 将案件按难度等级均匀分配至A组和B组的评估任务集中，确保两组案件难度分布无显著差异 | 研究团队 | 第5周 |

**实验操作流程（单人单次任务）：**

```
开始 → 登录实验平台 → 接收案件材料 → 阅读分析案件（计时开始）
   → A组：仅依赖个人经验进行分析判断
   → B组：参考AI分析报告 + 个人经验进行分析判断
   → 填写认定结论 → 提交（计时结束）→ 下一起案件
```

### 7.5 阶段五：统计分析阶段

| 步骤 | 内容 | 方法/工具 | 负责方 |
|------|------|-----------|--------|
| 1. 数据清洗 | 处理缺失值、异常值，检查数据完整性 | Python (pandas) / R | 数据分析师 |
| 2. 描述性统计 | 计算各组的基本统计量（均值、标准差、中位数等） | Python / R / SPSS | 数据分析师 |
| 3. 一致性分析 | 计算各组组内Kappa系数 | Python (scikit-learn) / R (irr) | 数据分析师 |
| 4. 耗时对比分析 | 两组平均耗时的差异检验 | 独立样本t检验 / Mann-Whitney U检验 | 数据分析师 |
| 5. AI一致率计算 | 计算AI结论与判决结果的吻合度 | Python | 数据分析师 |
| 6. 多因素分析 | 分析案件难度、从业年限等变量对结果的影响 | 方差分析(ANOVA) / 回归分析 | 数据分析师 |
| 7. 显著性检验 | 对主要假设进行统计显著性检验，显著性水平 α=0.05 | — | 数据分析师 |
| 8. 结果可视化 | 生成统计图表，直观呈现分析结果 | Python (matplotlib/seaborn) / R (ggplot2) | 数据分析师 |
| 9. 结论撰写 | 根据分析结果撰写实验结论报告 | — | 研究团队 |

**显著性检验标准：**

- 采用双尾检验（two-tailed test）
- 显著性水平设定为 **P < 0.05**，即当P值小于0.05时，认为组间差异具有统计学意义
- 对于多重比较，采用Bonferroni校正或其他适当的多重比较校正方法

2. 您需要独立完成20-30件已审结帮信罪案件的"主观明知"认定判断。
3. 每件案件需在实验平台上完成材料阅读、分析和结论提交。
4. 实验平台会自动记录您完成每件案件的分析耗时。
5. 预期总参与时间约为2-3周，分阶段安排，每次连续工作时间不超过2小时。

#### 三、潜在风险与不适

- **时间和精力消耗**：参与实验可能需要占用您的部分工作时间，但实验安排在合理范围内，不会对日常工作造成重大影响。
- **轻微认知疲劳**：连续进行案件分析可能导致疲劳，实验中设置了充分的休息间隔，您也可以随时申请暂停。

#### 四、数据使用与隐私保护

1. 您的基本背景信息（从业年限、专业领域等）将被收集用于研究分析。
2. 您的个人身份信息（姓名、联系方式等）将进行匿名化处理，研究报告中不会出现任何可识别个人身份的信息。
3. 所有实验数据仅用于本次研究，不会用于其他目的。
4. 实验案件材料均已进行脱敏处理，但您仍需签署保密协议，不得对外泄露案件内容。
5. 研究完成后，原始数据将按规定期限保存后销毁。
