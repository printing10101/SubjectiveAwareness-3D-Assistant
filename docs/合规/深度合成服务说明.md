# 深度合成服务说明

### 1.1 服务名称

### 1.2 服务类型

### 1.3 服务形态

## 二、深度合成技术应用说明

### 2.1 技术定义
本服务所称"深度合成"是指利用大语言模型（LLM）技术，对输入的法律案件文本进行深度理解、分析和结构化生成的过程。具体包括：

1. **文本理解**：对案件事实文本进行语义理解和关键信息提取
2. **知识推理**：基于法律知识和案例图谱进行逻辑推理
3. **内容生成**：生成结构化的分析报告和推理过程

### 2.2 技术原理

#### 2.2.1 大语言模型基础
- **模型架构**：Transformer架构
- **模型规模**：DeepSeek-R1-7B（70亿参数）
- **训练数据**：大规模法律文本语料库
- **推理能力**：具备法律领域的理解和推理能力

#### 2.2.2 两阶段深度合成
1. **事实提取阶段**：
   - 输入：原始案件文本
   - 处理：提取六要素（行为人、行为、工具、通讯、获利、辩解）
   - 输出：结构化事实JSON（约300 Token）

2. **维度分析阶段**：
   - 输入：结构化事实 + 法律知识
   - 处理：三维度深度分析（行为评估、认知匹配、辩解合理性）
   - 输出：分析报告JSON（约500 Token）

#### 2.2.3 知识增强机制
- **知识图谱**：三层法律知识图谱提供推理支撑
  - 要素图谱层：12个特征节点 + 15条关系边
  - 推理图谱层：6条推定规则 + 3条典型推理路径
  - 案例图谱层：25个贵州帮信罪案例
- **Prompt工程**：注入《帮信解释》第11条等法律知识
- **案例检索**：基于Jaccard相似度的类案推送

### 2.3 合成内容类型

#### 2.3.1 行为评估报告
- **合成内容**：
  - 客观行为异常度评分（0-10分）
  - 关键行为指标识别
  - 行为模式分析
  - 与典型作案模式的匹配度
- **示例输出**：
  ```json
  {
    "score": 8.5,
    "reasoning": "嫌疑人以明显低于市场价的价格大量收购银行卡，且交易方式隐蔽，使用暗语沟通，表现出明显的规避监管意图...",
    "key_indicators": [
      "交易价格异常（低于市场价50%）",
      "交易频率异常（单日交易20+次）",
      "使用暗语沟通（'卡片'代指银行卡）"
    ]
  }
  ```

#### 2.3.2 认知匹配报告
- **合成内容**：
  - 主观认知能力匹配度评分（0-10分）
  - 行为模式匹配分析
  - 通讯记录异常分析
  - 明知程度推断
- **示例输出**：
  ```json
  {
    "score": 7.0,
    "reasoning": "嫌疑人具有完全刑事责任能力，其行为模式与典型帮信罪作案模式高度匹配（匹配度85%），通讯记录中多次出现规避监管的表述...",
    "pattern_match": "与'低价大量收购银行卡'典型作案模式匹配度85%"
  }
  ```

#### 2.3.3 辩解合理性报告
- **合成内容**：
  - 辩解逻辑一致性评分（0-10分）
  - 辩解与客观证据矛盾点
  - 辩解合理性分析
  - 逻辑漏洞识别
- **示例输出**：
  ```json
  {
    "score": 3.0,
    "reasoning": "嫌疑人辩解'不知道是违法犯罪'与客观证据存在多处矛盾：1）使用暗语沟通表明其意识到行为不当；2）交易价格异常表明其知晓交易不正常...",
    "contradictions": [
      "辩解'不知情'与使用暗语沟通矛盾",
      "辩解'正常交易'与异常低价矛盾"
    ]
  }
  ```

#### 2.3.4 综合分析总结
- **合成内容**：
  - 三维度综合评分
  - 整体分析结论
  - 关键发现摘要
  - 建议关注点
- **示例输出**：
  ```json
  {
    "overall_summary": "综合分析显示，嫌疑人主观明知程度较高（综合评分7.5/10）。客观行为异常度8.5分，表现出明显的规避监管意图；认知匹配度7.0分，行为模式与典型作案模式高度匹配；辩解合理性仅3.0分，与客观证据存在多处矛盾。建议重点关注交易价格异常和暗语沟通两个关键点。"
  }
  ```

## 三、内容标识与管理机制

### 3.1 内容标识

#### 3.1.1 生成内容标识
所有由系统生成的分析内容均带有明确标识：

1. **元数据标识**：
   ```json
   {
     "generated_by": "帮信罪主观明知智能分析系统",
     "version": "1.0.0",
     "timestamp": "2026-06-13T10:30:00Z",
     "model": "deepseek-r1:7b",
     "analysis_id": "uuid-v4"
   }
   ```

2. **界面标识**：
   - 分析报告顶部显示"AI辅助分析结果"标识
   - 每个维度评分旁显示"AI生成"标签
   - 推理过程部分标注"AI推理"水印

3. **导出文件标识**：
   - PDF报告首页注明"本报告包含AI辅助分析内容"
   - 每页底部添加"AI生成内容，仅供参考"声明

#### 3.1.2 人工审查标识
经人工审查后的内容添加审查标识：
```json
{
  "reviewed_by": "检察官姓名",
  "reviewed_at": "2026-06-13T11:00:00Z",
  "review_status": "approved|rejected|modified",
  "review_comments": "审查意见"
}
```

### 3.2 内容管理机制

#### 3.2.1 内容生成管理
1. **生成流程控制**：
   - 输入内容长度校验（10-50000字符）
   - 输入内容格式校验（纯文本）
   - 复杂度自动判定与路由
   - 生成过程超时控制

2. **生成质量保障**：
   - 输出结构完整性校验
   - 评分范围校验（0-10分）
   - 推理过程逻辑性检查
   - 异常输出拦截与重试

3. **生成日志记录**：
   - 记录每次生成的输入、输出、耗时
   - 记录生成过程中的异常与重试
   - 日志保留期限：7天

#### 3.2.2 内容存储管理
1. **存储策略**：
   - 分析结果存储于数据库（SQLite/PostgreSQL）
   - 缓存结果存储于文件系统（7天TTL）
   - 敏感数据加密存储

2. **存储安全**：
   - 访问权限控制（基于角色）
   - 数据完整性校验
   - 定期备份与恢复测试

3. **存储清理**：
   - 过期数据自动清理（缓存7天）
   - 历史数据归档（按需）
   - 敏感数据销毁（按法规要求）

#### 3.2.3 内容使用管理
1. **使用权限控制**：
   - 基于角色的访问控制（RBAC）
   - API调用频率限制
   - 敏感操作审计日志

2. **使用追踪**：
   - 记录内容查看、下载、导出行为
   - 记录内容修改历史
   - 追踪内容传播路径

3. **使用声明**：
   - 每次使用时显示免责声明
   - 明确标注"辅助参考，不构成法律意见"
   - 要求人工审查确认

#### 3.2.4 内容修改管理
1. **修改权限**：
   - 仅授权用户可修改内容
   - 修改需记录修改人和修改原因
   - 保留修改历史版本

2. **修改流程**：
   - 提交修改申请
   - 审批流程（如需）
   - 执行修改
   - 记录修改日志

3. **修改审计**：
   - 记录所有修改操作
   - 定期审计修改记录
   - 异常修改告警

## 五、深度合成服务局限性

### 5.1 技术局限性
1. **理解能力局限**：
   - 模型对复杂法律概念的理解可能不准确
   - 对隐含信息的推理能力有限
   - 对新型作案模式的识别能力受限

2. **生成能力局限**：
   - 生成的推理过程可能不够严谨
   - 评分可能存在偏差
   - 对边缘案例的处理能力有限

3. **知识更新局限**：
   - 模型知识截止于训练时间
   - 新法律法规可能未及时纳入
   - 新司法解释可能未及时更新

### 5.2 应用局限性
1. **适用范围局限**：
   - 仅适用于帮信罪"主观明知"要素分析
   - 不适用于其他罪名分析
   - 不适用于量刑建议

2. **使用条件局限**：
   - 依赖输入文本质量
   - 需要人工审查确认
   - 不能替代司法判断

3. **准确性局限**：
   - 分析结论准确率约70%
   - 存在误判可能
   - 仅供参考

### 5.3 法律局限性
1. **法律效力局限**：
   - 分析结果不具有法律效力
   - 不能作为证据使用
   - 不能替代司法文书

2. **责任承担局限**：
   - 使用后果由使用者承担
   - 系统提供方不承担法律责任
   - 人工审查是必要环节

### 6.2 使用责任声明
使用本服务产生的任何后果由使用者自行承担。系统提供方不对使用结果承担任何责任。

### 6.3 内容准确性声明
本服务提供的分析结果仅供参考，可能存在误差。使用者应结合案件实际情况进行综合判断。

### 6.4 数据安全声明
本服务采用本地化部署，数据不出本机。但仍建议使用者注意数据保护，避免输入敏感个人信息。

### 6.5 服务变更声明
本服务可能随版本更新而变化，分析结果可能发生变化。使用者应关注版本更新说明。
