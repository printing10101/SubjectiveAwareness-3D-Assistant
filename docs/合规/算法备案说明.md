# 算法备案说明

## 一、算法基本信息

### 1.1 算法名称
帮信罪主观明知智能分析算法

### 1.2 算法版本
v1.0.0

### 1.3 算法类型
基于大语言模型的法律文本分析算法

### 1.4 应用场景
- 帮助信息网络犯罪活动罪（帮信罪）案件中"主观明知"要素的辅助分析
- 司法人员案件审查辅助决策
- 法律案件结构化分析与推理

### 1.5 服务对象
- 检察机关检察官
- 司法办案人员
- 法律研究人员

## 二、算法原理与技术架构

### 2.1 核心技术
本算法基于以下核心技术构建：

1. **大语言模型（LLM）**：采用 DeepSeek-R1-7B 开源模型，通过本地化部署确保数据安全
2. **Prompt 工程**：设计分级提示词模板，注入《帮信解释》第11条等法律知识
3. **两阶段推理管线**：根据案件复杂度自动选择单次推理或管线推理
4. **知识图谱增强**：三层知识图谱（要素图谱层、推理图谱层、案例图谱层）提供推理支撑

### 2.2 算法流程

```
输入案件文本
    ↓
复杂度判定（基于文本长度、行为人数量、关键词等）
    ↓
┌─────────────┬─────────────┐
│ 简单案件     │ 复杂案件     │
│ (<2000字)   │ (>=2000字)  │
└─────────────┴─────────────┘
    ↓              ↓
单次推理      两阶段推理
    ↓              ↓
    ↓         Stage 1: 事实提取
    ↓         (行为人/行为/工具/通讯/获利/辩解)
    ↓              ↓
    ↓         Stage 2: 维度分析
    ↓              ↓
    └──────┬───────┘
           ↓
    三维度分析结果
    - 行为评估（客观行为异常度）
    - 认知匹配（主观认知能力匹配度）
    - 辩解合理性（辩解逻辑一致性）
           ↓
    结构化输出（评分+推理过程+关键指标）
```

### 2.3 三维度分析模型

#### 维度一：行为评估（Behavior Assessment）
- **评估内容**：客观行为的异常程度
- **关键指标**：
  - 交易价格合理性
  - 交易方式合规性
  - 行为频率与规模
  - 规避监管行为
- **评分范围**：0-10分

#### 维度二：认知匹配（Cognitive Assessment）
- **评估内容**：主观认知能力与明知程度
- **关键指标**：
  - 行为人认知能力（年龄、教育、经验）
  - 行为模式与典型作案模式的匹配度
  - 通讯记录中的异常表述
- **评分范围**：0-10分

#### 维度三：辩解合理性（Defense Assessment）
- **评估内容**：辩解的逻辑一致性与合理性
- **关键指标**：
  - 辩解与客观证据的矛盾点
  - 辩解逻辑的自洽性
  - 辩解与常理的符合度
- **评分范围**：0-10分

## 三、数据处理流程

### 3.1 数据输入
- **输入类型**：纯文本（案件事实描述）
- **输入格式**：UTF-8编码文本
- **最小长度**：10字符
- **最大长度**：50,000字符
- **输入内容建议**：
  - 嫌疑人基本信息
  - 案件事实经过（时间、地点、行为）
  - 交易细节（价格、方式、频率）
  - 沟通记录（聊天内容、暗语使用）
  - 辩解内容
  - 相关证据线索

### 3.2 数据处理

#### 3.2.1 预处理阶段
1. **文本清洗**：去除特殊字符、标准化格式
2. **长度校验**：验证输入长度在允许范围内
3. **复杂度评估**：
   - 基于字符数的初步判定
   - 基于关键词密度的加权评估
   - 基于句子复杂度的综合分析
   - 综合评分分类（简单/中等/复杂）

#### 3.2.2 推理阶段
1. **Prompt 构建**：
   - 系统提示词：角色定义、任务说明、输出格式要求
   - 法律知识注入：《帮信解释》第11条摘要
   - 案件文本嵌入
2. **模型调用**：
   - 通过 Ollama API 发送推理请求
   - 配置参数：temperature=0.2, top_p=0.9, num_ctx=8192
   - 超时控制：动态超时策略（基于文本长度）
3. **结果解析**：
   - JSON 格式解析
   - 数据模型校验（Pydantic）
   - 异常处理与重试机制

#### 3.2.3 后处理阶段
1. **结果校验**：验证输出结构完整性
2. **缓存管理**：MD5摘要索引，7天TTL
3. **日志记录**：审计日志、性能指标

### 3.3 数据输出
- **输出格式**：JSON
- **输出内容**：

## 四、算法安全措施

### 4.1 数据安全
1. **本地化部署**：
   - 模型推理完全在本地进行
   - 数据不出本机，避免网络传输风险
   - 不依赖外部云服务

2. **数据加密**：
   - 敏感数据加密存储（AES-256）
   - 传输层加密（HTTPS/TLS）
   - 密码哈希存储（bcrypt，12轮）

3. **访问控制**：
   - JWT 双Token认证机制
   - 基于角色的权限控制（RBAC）
   - API 限流保护

4. **审计追踪**：
   - 完整的操作审计日志
   - 记录用户、IP、操作内容、时间戳
   - 日志保留期限：7天

### 4.2 算法安全
1. **输入验证**：
   - 严格的输入格式校验
   - 防止SQL注入（参数化查询）
   - 防止XSS攻击（输入转义）

2. **输出过滤**：
   - 结果数据模型校验
   - 敏感信息脱敏处理
   - 异常输出拦截

3. **模型安全**：
   - 使用开源模型，代码可审计
   - 模型文件完整性校验
   - 定期安全更新

### 4.3 系统安全
1. **网络安全**：
   - CORS 白名单配置
   - 速率限制（Rate Limiting）
   - 请求签名验证

2. **系统加固**：
   - 依赖包漏洞扫描
   - 代码安全审计（bandit）
   - 安全头配置（CSP、HSTS）

3. **应急响应**：
   - 错误追踪系统（Sentry）
   - 健康检查端点
   - 快速回滚机制

## 五、算法局限性与声明

### 5.1 算法局限性
1. **辅助性质**：本算法仅提供辅助分析参考，不构成法律意见
2. **准确率限制**：分析结论与专业检察官判断的一致性约70%
3. **适用范围**：仅适用于帮信罪"主观明知"要素的分析
4. **依赖输入质量**：分析结果受输入文本质量和完整性影响

### 5.2 使用声明
1. **人工审查**：所有分析结果必须经过司法人员人工审查
2. **不替代判断**：本系统不替代司法人员的独立判断
3. **责任承担**：使用本系统产生的任何后果由使用者自行承担
4. **持续改进**：算法将持续优化，分析结果可能随版本更新而变化
