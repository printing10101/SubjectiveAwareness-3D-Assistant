## 一、项目概览

### 1.1 项目目标
构建一套基于大语言模型的帮信罪主观明知智能分析系统，实现案件材料自动解析、三维度智能分析、可视化报告生成等核心功能。

### 1.2 技术栈
- **后端**: FastAPI + SQLAlchemy + SQLite/PostgreSQL
- **前端**: React 18 + TypeScript + Vite
- **AI模型**: Ollama (deepseek-r1:7b)
- **数据库**: SQLite (开发) / PostgreSQL (生产)

## 五、已知限制清单

### 5.1 技术限制

1. **暂不支持批量案件处理**
   - 原因: 当前架构设计为单案件串行分析，批量处理需要引入任务队列和并发控制机制
   - 临时方案: 通过脚本逐个导入案件
   - 预计解决: 2026-Q4

2. **暂不支持多模型并行对比**
   - 原因: Ollama本地部署资源有限，并行推理会导致显存不足
   - 临时方案: 串行执行多模型对比
   - 预计解决: 需升级硬件或引入模型服务集群

3. **暂不支持实时协作标注**
   - 原因: 当前标注系统为单人模式，多人协作需要引入WebSocket实时通信和冲突解决机制
   - 临时方案: 使用任务分配机制
   - 预计解决: 2026-Q4

4. **暂不支持自定义分析维度**
   - 原因: 三维度分析模型经过司法专家验证，自定义维度可能影响分析结果的法律效力
   - 临时方案: 无
   - 预计解决: 需与司法专家协商

5. **暂不支持离线部署**
   - 原因: 系统依赖Ollama服务进行模型推理，完全离线需要本地部署完整模型服务栈
   - 临时方案: 在内网部署Ollama服务
   - 预计解决: 需硬件支持

### 5.2 性能限制

1. **单案件分析响应时间**: 约30-60秒（取决于案件复杂度）
2. **并发处理能力**: 建议不超过5个并发请求
3. **数据库容量**: SQLite版本建议不超过10万条记录

## 七、给后续维护者的5条关键建议

### 7.1 代码维护

**建议**: 逐步完善类型注解，优先处理核心模块

**详细说明**:
- 当前mypy检查发现286个类型错误，主要集中在pipeline.py（78个）
- 建议按以下顺序修复：
  1. 核心类型定义 (app/types/)
  2. 服务层 (app/services/)
  3. 路由层 (app/routers/)
  4. 工具函数 (app/utils/)
- 每次修复后运行 `mypy --strict` 验证

### 7.2 性能优化

**建议**: 引入异步任务队列，提升批量处理能力

**详细说明**:
- 当前案件分析为同步处理，建议引入Celery或RQ
- 优化方向：
  1. 案件解析与分析分离
  2. 引入Redis作为任务队列
  3. 支持多Worker并行处理
  4. 添加任务优先级和进度追踪

### 7.3 功能扩展

**建议**: 建立插件化架构，支持自定义分析维度

**详细说明**:
- 当前三维度分析模型固定，扩展性不足
- 建议设计：
  1. 维度分析接口标准化
  2. 支持动态加载分析插件
  3. 建立维度配置管理界面
  4. 提供维度效果评估机制

### 7.4 安全加固

**建议**: 实施零信任安全架构，强化访问控制

**详细说明**:
- 当前安全措施已基础完善，建议进一步提升：
  1. 实施RBAC细粒度权限控制
  2. 添加操作审计日志分析
  3. 引入API网关统一认证
  4. 定期进行安全扫描和渗透测试
