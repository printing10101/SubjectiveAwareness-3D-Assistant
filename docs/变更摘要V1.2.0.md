# 帮信罪辅助裁定软件 V1.2.0 变更摘要

**版本**: V1.2.0  
**发布日期**: 2026-06-22  
**变更类型**: 次版本更新（向下兼容的功能增强与架构优化）

---

## 一、变更目的

本次版本更新旨在对 V1.1.0 进行系统性架构优化和代码质量提升，主要目标包括：

1. **模块化重构**: 将巨型文件拆分为可维护的模块结构，降低代码耦合度
2. **服务整合优化**: 合并功能重叠的服务模块，消除冗余代码
3. **前端组件化**: 将大型视图文件拆分为独立组件，提升可复用性
4. **代码质量提升**: 清理冗余注释、统一代码风格、优化导入路径
5. **文档完善**: 补充清理计划、重构映射、设计文档等技术资料

---

## 二、主要变更内容

### 2.1 后端架构重构

#### 2.1.1 Pipeline 模块化拆分

**变更说明**: 将原 `pipeline.py`（3114 行）拆分为独立的 `pipeline/` 子包

**新增文件**:
- `backend/app/services/pipeline/__init__.py` - 包入口，导出 V1/V2 协议接口
- `backend/app/services/pipeline/orchestrator.py` - 主流程编排器，实现 10 阶段分析流程
- `backend/app/services/pipeline/complexity.py` - 复杂度分类模块
- `backend/app/services/pipeline/knowledge.py` - 知识检索模块（Neo4j/SQLite FTS/内存兜底）
- `backend/app/services/pipeline/v1_analysis.py` - V1 协议实现（0-10 分评分）
- `backend/app/services/pipeline/v2_protocol.py` - V2 协议实现（档级 + 规则/标签/冲突）
- `backend/app/services/pipeline/json_utils.py` - JSON 处理工具

**V2 协议核心特性**:
- 三维度分析架构：事实审查 → 模式匹配 → 矛盾分析
- 4×4×4 档级组合算法：64 种组合的确定性映射
- 规则权重升级/抗辩降档机制

**向后兼容**: 保留原 `pipeline.py` 作为兼容层，内部委托给 `pipeline/` 子包
