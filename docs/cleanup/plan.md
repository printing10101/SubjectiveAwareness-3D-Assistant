# V1.1.0 → V1.2.0 Cleanup Plan

**文档版本**: 1.0  
**创建日期**: 2026-06-17  
**文档目的**: 规划 V1.1.0 到 V1.2.0 的系统清理工作，不包含任何业务代码修改

---

## 1. 现状分析

### 1.1 后端服务模块
- **服务模块数量**: 41个 services
- **核心文件规模**: `pipeline.py` 代码规模达到 3114 行
- **模块复杂度**: 存在大量功能重叠和职责不清的服务模块

### 1.2 前端组件冗余
- **重复组件**: 存在 2 套 Header/Sidebar 组件
  - 第一套位于 `components/common/` 目录
  - 第二套位于 `components/layout/` 目录
- **组件使用混乱**: 不同页面引用不同套件的组件，导致维护困难

### 1.3 样式文件冲突
- **CSS 文件总数**: 6 个主要样式文件
- **重复文件对**:
  - `base.css` 与 `global.css` 内容重复
  - `tokens.css` 与 `variables.css` 内容重复
- **样式冲突**: 重复定义导致样式覆盖问题，增加调试成本

### 1.4 前端文件复杂度
- **巨型文件数量**: 5 个 view 文件
- **代码规模**: 单文件代码量均超过 1000 行
- **维护难度**: 文件过大导致难以理解、测试和维护

---

## 2. 目标指标

### 2.1 代码文件规模控制
- **目标**: 单文件最大行数严格控制在 ≤800 行
- **衡量标准**: 所有源文件（.py, .vue, .ts, .js）均不超过 800 行
- **验收方法**: 自动化脚本扫描统计文件行数

### 2.2 后端架构优化
- **目标**: services 服务模块数量从 41 个收敛至约 25 个
- **优化方向**: 
  - 合并功能相似的服务模块
  - 删除废弃未使用的服务
  - 提取通用逻辑到基础类
- **验收方法**: 统计 `backend/app/services/` 目录下模块数量

### 2.3 样式资源整合
- **目标**: CSS 文件数量从 6 个精简至 3 个
- **整合策略**:
  - 合并 `base.css` 和 `global.css` 为统一的 `base.css`
  - 合并 `tokens.css` 和 `variables.css` 为统一的 `tokens.css`
  - 保留 `components.css` 作为组件专用样式
- **验收方法**: 统计 `frontend/src/styles/` 目录下文件数量

### 2.4 前端文件拆分
- **目标**: 巨型 view 文件数量（>1000 行）减少至 0 个
- **拆分策略**:
  - 提取可复用组件到独立文件
  - 按功能模块拆分子组件
  - 使用组合式 API 重构逻辑
- **验收方法**: 自动化脚本扫描统计超过 1000 行的文件数量

### 2.5 代码总量优化
- **目标**: 实现净减少代码量约 7000 行
- **优化来源**:
  - 删除重复组件代码：约 1500 行
  - 合并重复 CSS 文件：约 800 行
  - 删除废弃服务模块：约 2000 行
  - 重构优化冗余逻辑：约 2700 行
- **验收方法**: 对比清理前后代码总行数
