# 用户手册

## 1. 系统简介

帮信罪主观明知分析系统是一款基于大语言模型（LLM）和知识图谱技术的司法智能辅助工具，专用于帮助信息网络犯罪活动罪（帮信罪）案件中的"主观明知"要素分析。系统从**客观行为异常度**、**认知能力与作案模式匹配度**、**辩解合理性**三个维度进行综合评估，辅助检察官、法官等司法人员进行标准化案件分析。

### 1.1 核心功能

| 功能模块 | 说明 |
|---------|------|
| 智能分析 | 输入案件事实文本，AI 自动进行三维度分析并生成分析报告 |
| 智能阅卷 | 上传 PDF/DOCX 文档，自动提取文本和实体信息 |
| 类案推送 | 基于知识图谱的相似案例推荐 |
| 量刑辅助 | 基于历史判例的量刑参考建议 |
| 知识图谱 | 法律法规与案例的知识网络查询 |
| 实验系统 | 支持回溯性对比实验的数据采集 |
| 案件管理 | 案件的创建、查看、搜索和删除 |

### 1.2 系统角色

| 角色 | 权限说明 |
|------|---------|
| 管理员 | 全部功能权限，包括用户管理、法律规则管理、数据导出 |
| 普通用户 | 分析功能、案件管理、报告查看、知识图谱查询 |

## 2. 快速上手

### 2.1 访问系统

启动服务后，在浏览器中访问：

- **开发环境**: `http://localhost:5173`
- **生产环境**: 根据部署配置的域名访问

### 2.2 登录系统

1. 打开系统首页，进入登录页面
2. 输入用户名和密码（默认管理员: admin / admin123）
3. 点击"登录"按钮
4. 登录成功后自动跳转到分析主页面

如果登录失败，请检查：
- 用户名和密码是否正确（注意大小写）
- 账户是否已被管理员禁用
- 后端服务是否正常运行

### 2.3 系统导航

登录后，页面顶部显示导航栏，包含以下功能入口：

| 导航项 | 对应页面 | 说明 |
|--------|---------|------|
| 分析主页 | `/main` | 核心分析功能，输入案件文本并触发 AI 分析 |
| 智能阅卷 | `/review` | 上传文档、文本编辑、实体信息抽取 |
| 实验采集 | `/experiment` | 回溯性对比实验的案例分配与判断提交 |
| 案件管理 | `/cases` | 案件的列表查看、搜索、创建和删除 |
| 系统管理 | `/settings` | 法律规则、模型版本、系统日志、用户管理 |

### 3.2 操作步骤

#### 步骤一：输入案件事实

**方式1：手动输入**
- 在"案件事实输入"文本框中直接输入或粘贴案件事实描述
- 文本长度限制：10 - 50,000 字符
- 建议包含：交易过程、聊天记录、嫌疑人行为、获利情况等关键信息

**方式2：使用 Demo 案例**
- 点击页面中的 Demo 案例按钮（"明显明知"、"边缘情况"、"确实不明知"三种类型）
- 系统会自动填充预设的案例文本到输入框
- Demo 案例已预缓存，首次分析可立即返回结果

**方式3：通过智能阅卷上传文档**
- 先进入"智能阅卷"页面上传文档提取文本
- 提取完成后可导航回分析主页，文本会自动带入

#### 步骤二：开始分析

1. 确认案件文本已输入（至少 10 个字符）
2. 点击"开始分析"按钮（渐变色按钮）
3. 等待 AI 分析完成，期间会显示加载动画
4. 分析完成后自动跳转到报告页面

#### 步骤三：查看分析详情

分析完成后，可在页面底部展开"分析详情"查看：
- **缓存状态**: 显示分析结果是否来自缓存（绿色=缓存命中，红色=首次分析）
- **响应时间**: 本次请求的总处理时间（毫秒）
- **Token 估算**: 输入和输出的 Token 数量估算

### 3.3 案例分析维度

系统从以下三个维度进行分析：

| 维度 | 说明 | 评分范围 |
|------|------|---------|
| 客观行为异常度 | 评估行为是否偏离正常交易/通讯模式 | 0（正常） - 10（极度异常） |
| 认知能力与作案模式匹配度 | 评估行为人认知水平与典型犯罪模式的匹配程度 | 0（无匹配） - 10（高度匹配） |
| 辩解合理性 | 评估嫌疑人辩解的逻辑合理性和可信度 | 0（完全不合理） - 10（完全合理） |

**综合评分（knowledge_score）**：0-10 分
- 0-3 分：确实不明知
- 4-6 分：可能不明知或可能明知（边缘情况）
- 7-10 分：明显明知

### 4.2 文档上传

1. **拖放上传**: 将 PDF、DOCX 或 DOC 文件拖放到上传区域
2. **点击选择**: 点击上传区域，从文件选择器中选择文件
3. **格式支持**: PDF（含自动 OCR）、DOCX、DOC
4. **大小限制**: 单文件最大 20MB

上传流程：
```
选择文件 → 上传校验（格式+大小）→ 文件上传（显示进度条）→
文本提取（显示进度）→ 文本自动填入编辑器
```

### 4.3 文本编辑

文档文本提取后自动填入富文本编辑器，支持：
- **格式化**: 加粗（B）、斜体（I）、下划线（U）
- **清除格式**: 一键移除选中文本的格式
- **清空内容**: 清空编辑器全部内容
- **字数统计**: 实时显示文本字符数

### 4.4 实体信息抽取

1. 在编辑器中确认或调整案件文本
2. 点击"开始抽取"按钮
3. 系统自动识别并展示以下类型实体：

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| 人员信息 | 嫌疑人、被害人、证人等 | 张三、李四 |
| 银行卡信息 | 涉案银行账户 | 工商银行6222xxxx |
| 通讯工具 | 涉案通讯方式 | 微信、QQ、Telegram |
| 交易信息 | 金额、时间、平台 | 50万元、2023年1月 |
| 关系信息 | 实体之间的关联关系 | 张三→转账→李四 |

抽取结果可进行编辑：
- 修改实体的值
- 删除不需要的实体
- 手动添加自定义实体
- 查看实体关系图

### 4.5 进入分析

在智能阅卷页面完成文本确认和实体抽取后，可点击"开始分析"按钮直接进入分析流程，系统将自动跳转到分析报告页面。

## 6. 分析报告

分析报告页面展示 AI 分析的三维度结果。

### 6.1 报告内容

报告包含以下部分：

**综合结论**
- 结论类型（明显明知 / 边缘情况 / 确实不明知）
- 总体结论描述
- 置信度百分比

**维度分析**
- 每个维度的评分（0-100 分）
- 推理依据文字说明
- 引用证据列表
- 相关法条引用

**推理链条**
- 从证据到规则的完整逻辑路径
- 多链条展示（如适用）

### 6.2 报告操作

- **复制报告**: 点击"复制"按钮，将报告导出为 Markdown 格式并复制到剪贴板
- **导出 PDF**: 点击"导出 PDF"，使用 html2canvas + jsPDF 生成本地 PDF 文件
- **返回**: 点击返回按钮回到分析主页

## 7. 知识图谱

知识图谱功能通过 REST API 提供，用户可在系统管理页面查看或通过 API 直接查询。

### 7.1 查询功能

| 查询类型 | 说明 |
|---------|------|
| 规则匹配 | 根据证据类型匹配适用的法律推定规则 |
| 相似案例 | 根据案件特征检索相似案例 |
| 推理追溯 | 追踪特定结论的完整推理路径 |

### 7.2 应用场景

- **证据评估**: 输入识别到的异常行为特征，系统返回对应的法律推定规则
- **类案参考**: 输入案件关键特征，获取历史相似案例及判决结果
- **论证辅助**: 追溯特定法律结论的完整推理链条，辅助撰写法律文书

## 9. 实验系统

实验系统支持回溯性对比实验的数据采集，用于评估 AI 辅助对司法判断的影响。

### 9.1 实验流程

```
实验者登录 → 随机分组（A组/无AI辅助 / B组/有AI辅助）→
分配案例 → 阅读案件 → 提交判断 → 下一案例 → 数据导出
```

### 9.2 功能说明

| 功能 | 说明 |
|------|------|
| 案例分配 | 随机从实验案例集中分配案例给实验者 |

| 判断提交 | 记录明知认定、置信度、推理文本、反应时间 |
| AI 采纳标记 | B 组实验者标记是否采纳 AI 建议 |
| 进度追踪 | 实时查看已完成和待完成的案例数量 |
| 数据导出 | 管理员可导出完整实验数据用于统计分析 |

## 10. 常见问题

### 10.1 分析结果不准确

- 确认案件文本包含足够的案件事实细节
- 建议提供交易过程、聊天记录、嫌疑人供述等关键信息
- 系统分析结果仅供参考，不构成法律意见

### 10.2 分析速度慢

- 首次分析（缓存未命中）需要调用 LLM，通常需要 30-60 秒
- 复杂案件（文本 > 2000 字或行为人 > 3）会触发两阶段推理管线
- 同类案件再次分析时将命中缓存，响应时间 < 1 秒

### 10.3 文档上传失败

- 确认文件格式为 PDF、DOCX 或 DOC
- 确认文件大小不超过 20MB
- 确认文件未损坏

### 10.4 登录失败

- 确认已启动后端服务和数据库
- 确认用户名和密码正确
- 管理员可在系统管理页面重置用户密码

## 11. 注意事项

1. **法律免责**: 系统所有分析结果均由 AI 模型生成，仅供参考，不构成法律意见。最终司法判断应由专业司法人员作出
2. **数据安全**: 系统默认使用 SQLite 本地数据库，敏感案件数据建议部署在安全环境中
3. **模型局限性**: 系统针对帮信罪场景优化，不适用于其他类型的案件分析
4. **网络要求**: AI 推理依赖本地 Ollama 服务，无需互联网连接
