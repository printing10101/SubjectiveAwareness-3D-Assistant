## 三、相关背景与现有技术

### 3.1 司法背景

**帮信罪**（帮助信息网络犯罪活动罪，《中华人民共和国刑法》第二百八十七条之二）是 2015 年《刑法修正案（九）》增设、2020 年《刑法修正案（十一）》进一步完善的罪名，规定：

> "明知他人利用信息网络实施犯罪，为其犯罪提供互联网接入、服务器托管、网络存储、通讯传输等技术支持，或者提供广告推广、支付结算等帮助，情节严重的，处三年以下有期徒刑或者拘役，并处或者单处罚金。"

自 2020 年以来，帮信罪已成为**年起诉量最大**的刑事案件之一。"主观明知"是该罪认定的**核心难点**——司法解释虽列举了若干"可推定明知"情形，但实务中：
- 嫌疑人常以"不知道""未明示"为由辩解；
- 证据多为非结构化电子数据，人工审阅效率低；
- 不同办案人员对相同证据的认定尺度不一；
- 资深检察官经验难以沉淀、传递。

### 3.2 现有技术不足

| 现有方案 | 不足 |
|----------|------|
| 关键词类案检索 | 仅字面匹配，缺语义理解；缺量化评分 |
| 单一规则引擎 | 规则静态，难应对新型犯罪；缺反证处理 |

## 四、发明内容

### 4.1 解决的技术问题

1. 如何将"主观明知"这一抽象法律要件，**结构化、可计算、可追溯**地建模；
2. 如何将大语言模型语义理解能力与法律知识图谱领域约束结合，**避免事实幻觉、提高可解释性**；
3. 如何在**本地化部署**前提下，完成文档解析、评分、比对、冲突检测、报告生成全流程；
4. 如何处理"明知"认定中**多路径矛盾、证据强度差异、辩解反证**等复杂情形。

### 4.2 核心创新点

#### 创新点 1：提出"三维度主观明知分析模型"

将"主观明知"分解为**三个正交可计算的维度**：
- **D1 客观行为异常度**（Behavior Assessment）：交易价格/频率/对象/行为规避四个子项；
- **D2 认知能力匹配度**（Cognitive Assessment）：年龄/职业/学历/经验 + 类案作案模式比对；
- **D3 辩解合理性**（Defense Assessment）：语义一致性/客观证据一致性/内部逻辑一致性三重校验。

每个维度均输出 **0-10 分量化评分 + 推理过程文本 + 关键依据列表**，支持**可解释、可审计**。

#### 创新点 2：构建"三层知识图谱约束机制"

设计**特征层 + 规则层 + 案例层**的三层知识图谱：
- **特征层**：案件事实特征节点（价格/频率/对象/行为/沟通）；
- **规则层**：依据《帮信解释》第十一条构建的"可推定明知"规则节点；
- **案例层**：真实判决案例节点，关联判决书来源、判决结果、量刑。

通过 **Jaccard 相似度 + 语义嵌入双通道** 进行类案检索，返回 Top-K 相似案例与对应推定规则，**有效降低大模型事实幻觉**。

#### 创新点 3：提出"三级冲突仲裁机制"

针对三维度评分内部矛盾、知识图谱多路径冲突，**首次**在"明知"分析任务中引入**"证据强度优先 → 规则权威性 → 路径可解释性"**三级仲裁策略：
- **第一级**：证据强度优先——矛盾时优先采纳客观证据支撑维度；
- **第二级**：规则权威性——冲突时优先采纳司法解释明文规定；
- **第三级**：路径可解释性——权重相当时选择可解释性更强的路径。

#### 创新点 4：实现"证据全链路溯源"

每个评分、每条结论**向上回溯至原始证据片段**，形成证据→评分→结论的完整溯源链，以可视化证据溯源图嵌入报告，**支持事后审计与责任认定**。

#### 创新点 5：实现"本地化司法 AI 部署"

通过 **Ollama + Qwen2.5/DeepSeek-R1 等开源大模型**，实现**全流程本地推理**，案件数据不出本机/本局域网，**满足司法数据安全要求**；支持国密 SM2/SM4 算法加密存储敏感字段。

#### 4.3.2 "主观明知"动态建模方法 6 步流程

**S1：案件事实结构化抽取**
- 输入：案件文本（200-20000 字符）或 PDF/DOCX
- 处理：文档解析 + LLM 第一阶段事实抽取
- 输出：结构化 JSON（6 大字段：嫌疑人/行为/交易/沟通/辩解/证据）

**S2：三层证据强度评估**
- 输入：S1 输出 + 证据项分解
- 处理：4 子维度（直接/间接/完整性/矛盾度）0-10 分量化
- 输出：证据强度分布矩阵 + 强度标签（强/中/弱/缺位）

**S3：三维度"明知"分析**
- 输入：S1 事实 + S2 证据矩阵
- 处理：3 个维度并行评分
- 输出：每维度 0-10 分 + 推理过程 + 关键依据

**S4：知识图谱约束与类案匹配**
- 输入：S1 事实
- 处理：Jaccard 相似度 + 语义嵌入双通道检索
- 输出：Top-K 相似案例 + 推定规则

**S5：冲突检测与多路径仲裁**
- 输入：S3 评分 + S4 检索结果
- 处理：三级仲裁
- 输出：最终推理路径 + 冲突说明

**S6：证据溯源与可解释报告生成**
- 输入：S1-S5 全部输出
- 处理：建立证据→评分→结论溯源链
- 输出：JSON/HTML/PDF 报告（含 7 大模块）

#### 4.3.3 核心算法伪代码

```python
def Analyze_Subjective_Knowledge(case_text):
    # S1 事实抽取
    facts = LLM_Extract_Facts(case_text)
    # S2 证据强度评估
    evidence_matrix = Evidence_Strength_Layer(facts)
    # S3 三维度评分（并行）
    D1_score, D1_reasoning = Behavior_Assessment(facts, evidence_matrix)
    D2_score, D2_reasoning = Cognitive_Assessment(facts, evidence_matrix, knowledge_graph)
    D3_score, D3_reasoning = Defense_Assessment(facts, evidence_matrix)
    # S4 知识图谱类案匹配
    similar_cases = Knowledge_Graph_Retrieve(facts, top_k=5)
    rules_matched = Rule_Matching(facts, knowledge_graph)
    # S5 冲突检测与仲裁
    conflicts = Conflict_Detector(D1_score, D2_score, D3_score, similar_cases)
    final_path = Arbitrate(conflicts, evidence_matrix)
    # S6 报告生成
    report = Generate_Report(D1, D2, D3, similar_cases, conflicts, final_path, evidence_matrix)
    return report
```
