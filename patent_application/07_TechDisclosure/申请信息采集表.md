## 四、发明技术信息

### 4.1 技术领域

人工智能 × 法律科技 × 司法智能化 — 帮信罪主观明知动态建模与辅助分析

### 4.2 关键创新点（建议 3-7 个，按重要性排序）

1. **"主观明知三维度分析模型"**（行为异常度 / 认知匹配度 / 辩解合理性）
2. **"三层知识图谱约束机制"**（特征-规则-案例）
3. **"三级冲突仲裁策略"**（证据强度→规则权威性→路径可解释性）
4. **"证据全链路溯源方法"**
5. **"本地化司法 AI 部署架构"**
6. **"两阶段推理管线 + Prompt 工程"**
7. **"知识库生命周期管理机制"**（规则、案例、模型版本化）

### 4.3 技术方案关键词

- 大语言模型（LLM）：Qwen2.5 / DeepSeek-R1 / Llama
- 本地推理：Ollama
- 知识图谱：Neo4j
- 后端：FastAPI（Python 3.10+）
- 前端：Vue 3 + Vite + TypeScript
- 数据库：SQLite / PostgreSQL
- 框架：Vue Router、Pinia、Axios
- 评分：0-10 分量化、推理过程文本、关键依据列表
- 部署：本地化、单 GPU（≥ 8GB）、微服务、Kubernetes 可选

### 4.4 解决的技术问题

1. 主观明知要件抽象、难以结构化；
2. 大语言模型事实幻觉；
3. 司法数据出网安全风险；
4. 边缘案件与多路径冲突无自动化处理机制。
