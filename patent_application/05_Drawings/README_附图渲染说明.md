## 附图源文件格式

本目录所有 `.mmd` 文件均为 **Mermaid 源文件**。Mermaid 是一种基于文本的图表描述语言，可被多种工具渲染为 PNG / SVG / PDF 等图片格式。

## 如何将 Mermaid 源文件渲染为图片

### 方式一：使用 Mermaid Live Editor（推荐，无需安装）

1. 打开浏览器访问 [https://mermaid.live/](https://mermaid.live/)；
2. 将对应 `.mmd` 文件中的内容**完整复制**到左侧代码编辑区；
3. 右侧将自动渲染出图表；
4. 点击右上角 **"PNG"** 或 **"SVG"** 按钮下载图片；
5. 将下载的图片**重命名**为对应的图号文件名，例如 `图1_系统五层架构总览图.png`；
6. 重复以上步骤，渲染全部 7 张图。

### 方式二：使用 Mermaid CLI（命令行方式）

#### 安装

```bash
# 使用 npm 安装
npm install -g @mermaid-js/mermaid-cli

# 验证安装
mmdc --version
```

#### 渲染为 PNG

```bash
# 进入附图目录
cd patent_application/05_Drawings

# 批量渲染所有 .mmd 文件为 PNG（推荐分辨率 -s 3 即 3 倍清晰度）
mmdc -i 图1_系统五层架构总览图.mmd -o 图1_系统五层架构总览图.png -w 2400 -H 1800 -b white -s 3
mmdc -i 图2_主观明知动态建模方法流程图.mmd -o 图2_主观明知动态建模方法流程图.png -w 2400 -H 3200 -b white -s 3
mmdc -i 图3_三维度分析模型结构图.mmd -o 图3_三维度分析模型结构图.png -w 2400 -H 1800 -b white -s 3
mmdc -i 图4_三层知识图谱结构图.mmd -o 图4_三层知识图谱结构图.png -w 2400 -H 1800 -b white -s 3
mmdc -i 图5_证据强度评估层结构图.mmd -o 图5_证据强度评估层结构图.png -w 2400 -H 1800 -b white -s 3
mmdc -i 图6_冲突检测与仲裁机制流程图.mmd -o 图6_冲突检测与仲裁机制流程图.png -w 2400 -H 2200 -b white -s 3
mmdc -i 图7_系统整体工作流时序图.mmd -o 图7_系统整体工作流时序图.png -w 2400 -H 2200 -b white -s 3
```

#### 渲染为 SVG（矢量图，缩放无损）

```bash
mmdc -i 图1_系统五层架构总览图.mmd -o 图1_系统五层架构总览图.svg
# ... 同上替换 i/o 参数
```

### 方式三：使用 VS Code + Mermaid 插件

1. 在 VS Code 中安装 **"Markdown Preview Mermaid Support"** 插件；
2. 打开任一 `.mmd` 文件；
3. 在编辑器右上角点击预览图标，即可看到渲染结果；
4. 通过截图或导出功能保存为 PNG。

### 方式四：使用 Typora / Obsidian 等 Markdown 编辑器

上述编辑器均支持 Mermaid 渲染，直接打开 `.mmd` 文件即可预览，再截图保存为 PNG。

## 提交要求

向国家知识产权局提交专利申请时：

1. **说明书附图**应使用 **PNG 或 TIFF** 格式，分辨率不低于 **300 DPI**；
2. 每张附图应占据**单独一页 A4 纸**，四周留白 ≥ 2.5cm；
3. 每张附图顶部应标注**图号与图名**，例如"图 1 系统五层架构总览图"；
4. 附图中**不应包含文字说明、注释或商业宣传**内容；
5. 附图序号应与说明书中的"附图说明"部分严格对应。
