# 版本号管理规范

## 1. 当前版本

**V1.0.0**

## 2. 版本号格式

采用 **语义化版本控制**（Semantic Versioning 2.0.0）：

```
主版本号.次版本号.修订号

示例：V1.0.0
```

- **主版本号**（Major）：当做了不兼容的 API 或功能修改时递增
- **次版本号**（Minor）：当做了向下兼容的功能性新增时递增
- **修订号**（Patch）：当做了向下兼容的问题修正时递增

## 3. 版本号标记位置

所有版本号必须保持统一，变更时同步更新以下所有位置：

| 位置 | 文件路径 | 配置字段 |
|------|----------|----------|
| 后端项目配置 | `backend/pyproject.toml` | `[project].version` |
| 前端项目配置 | `frontend/package.json` | `version` |
| Commitizen 配置 | `.cz.toml` | `[tool.commitizen].version` |
| FastAPI 应用 | `backend/app/main.py` | `app = FastAPI(version=...)` |
| API 参考文档 | `docs/api_reference.md` | API 响应中的 `version` 字段 |
| 发布检查清单 | `docs/release_checklist.md` | 版本号字段 |
| 项目准备报告 | `docs/project_readiness_report.md` | 版本号字段 |
| 综合技术文档 | `docs/综合技术文档.md` | 文档版本字段 |
| 合规文档 | `docs/合规/*.md` | 文档版本字段 |
| 报告导出模块 | `backend/app/services/report_generator.py` | `"version"` 字段 |
| 报告导出模块 | `backend/app/services/report_exporter.py` | `version` 默认值 |
| 报告 API 路由 | `backend/app/routers/reports.py` | `version` 参数 |
| 前端报告显示 | `frontend/src/views/ReportView.vue` | 报告中显示的版本号 |

## 4. 版本号变更规则

### 4.1 变更流程

1. **确定变更类型**：根据变更内容确定是主版本号、次版本号还是修订号递增
2. **更新所有标记位置**：按照第 3 节的列表逐一更新
3. **更新变更日志**：记录版本变更内容
4. **提交并打 Tag**：使用 `git tag vX.Y.Z` 标记版本

### 4.2 递增规则

| 变更类型 | 递增说明 | 示例 |
|----------|----------|------|
| **修订号** | Bug 修复、性能优化、代码重构（向下兼容） | V1.0.0 → V1.0.1 |
| **次版本号** | 新增功能、新增 API、新增配置（向下兼容） | V1.0.0 → V1.1.0 |
| **主版本号** | 不兼容的 API 变更、重大架构重构 | V1.0.0 → V2.0.0 |

### 4.3 特殊规则

- **预发布版本**：在版本号后添加 `-alpha.N`、`-beta.N`、`-rc.N`，如 `V1.0.0-beta.1`
- **构建元数据**：在版本号后添加 `+build.N`，如 `V1.0.0+build.20260601`

## 5. 版本号管理工具

项目使用 **Commitizen** 管理版本号（配置见 `.cz.toml`）：

```bash
# 查看当前版本
cz version

# 升级版本（自动更新 pyproject.toml 并创建 changelog）
cz bump
# 指定版本类型
cz bump --increment MAJOR
cz bump --increment MINOR
cz bump --increment PATCH
```

> **注意**：Commitizen 的 `version_provider = "pep621"` 从 `pyproject.toml` 读取版本号，因此 `pyproject.toml` 是版本号的唯一真实来源（Single Source of Truth）。

## 6. 一致性检查

每次版本发布前，必须执行以下检查：

```bash
# 检查所有位置版本号是否一致
# 后端版本
grep 'version = "' backend/pyproject.toml
# 前端版本
grep '"version"' frontend/package.json
# Commitizen 版本
grep 'version = "' .cz.toml
# FastAPI 版本
grep 'version="' backend/app/main.py
```
